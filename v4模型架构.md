# 模型架构：FaithDiff-style Dual-Side Alignment

## 1. Stage 1 训练流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Stage 1 Training Flow                              │
│                                                                             │
│  ┌──────────┐                                                    ┌────────┐ │
│  │  GT Image │──→ VAE.encode ──→ z_hq ──→ add_noise(t=200) ──→ x_hq_t  │ │
│  └──────────┘                                                    └───┬────┘ │
│                                                                       │      │
│  ┌──────────┐                              ┌─────────────────────┐    │      │
│  │  LQ Image │──→ RM ──→ VAE.encode ──→ z_lq│                     │    │      │
│  └──────────┘                              │  AlignmentModule    │    │      │
│                                            │                     │    │      │
│                                            │  concat(z_lq,x_hq_t)│←───┤      │
│                                            │       ↓             │    │      │
│                                            │  Encoder (SwinBlocks)│   │      │
│                                            │       ↓             │    │      │
│                                            │  Transformer (Attn) │    │      │
│                                            │       ↓             │    │      │
│                                            │  proj_out (zero_init)│   │      │
│                                            │       ↓             │    │      │
│                                            │  x_hq_t + delta     │──→ │      │
│                                            │  (= aligned)        │    │      │
│                                            └─────────────────────┘    │      │
│                                                                       ↓      │
│  ┌──────────────────────┐                                     ┌──────────┐  │
│  │  Frozen UNet + LoRA  │                                     │  noise   │  │
│  │                      │←── text_embed (CLIP)                │          │  │
│  │  conv_in(aligned)    │←── t=200                           │  randn   │  │
│  │       ↓              │                                     │          │  │
│  │  Down/Mid/Up blocks  │                                     └────┬─────┘  │
│  │       ↓              │                                          │        │
│  │  noise_pred          │                                          │        │
│  └──────────┬───────────┘                                          │        │
│             │                                                      │        │
│             └──────────────────→ L1 Loss ←─────────────────────────┘        │
│                         (noise_pred, noise)                                │
│                                                                             │
│  Trainable: AlignmentModule only (≈1.7M params)                            │
│  Frozen:    UNet + LoRA + VAE + RM + TextEncoder                           │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 训练数据流详细步骤

| 步骤 | 操作 | 维度变化 | 模块 |
|------|------|----------|------|
| 1 | GT: pixel → VAE.encode → z_hq | `[B,3,H,W]` → `[B,4,H/8,W/8]` | VAE (frozen) |
| 2 | z_hq + noise → x_hq_t (t=200) | `[B,4,H/8,W/8]` | DDPM Scheduler |
| 3 | LQ → RM → VAE.encode → z_lq | `[B,3,H,W]` → `[B,4,H/8,W/8]` | RM + VAE (frozen) |
| 4 | concat(z_lq, x_hq_t) | `[B,8,H/8,W/8]` | AlignmentModule |
| 5 | Encoder: 8→128→256→512 | `[B,512,H/8,W/8]` | ResidualSwinBlock ×3 |
| 6 | Transformer: spatial attention | `[B,512,H/8,W/8]` | ResidualAttentionBlock ×2 |
| 7 | proj_out: 512→4 (zero_init) | `[B,4,H/8,W/8]` | Conv2d |
| 8 | residual: x_hq_t + delta | `[B,4,H/8,W/8]` | — |
| 9 | UNet(aligned, t, text_embed) → noise_pred | `[B,4,H/8,W/8]` | UNet (frozen) |
| 10 | L1(noise_pred, noise) → backward | scalar | — |

### 推理与训练的关键差异

| 方面 | 训练 | 推理 |
|------|------|------|
| x_hq_t 来源 | GT → VAE.encode → add_noise | z_lq → add_noise（无GT近似） |
| z_lq 来源 | RM(LQ) → VAE.encode | RM(LQ) → tiled VAE.encode |
| UNet 后处理 | 无（仅预测噪声） | scheduler.step → z_pred → VAE.decode |
| 输出 | noise_pred (latent) | pixel-space HQ image |
| 大图处理 | 无需（crop 512×512） | tiled inference |

## 3. AlignmentModule 详细架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                     AlignmentModule Architecture                    │
│                                                                     │
│  Input: z_lq [B,4,H/8,W/8],  x_hq_t [B,4,H/8,W/8]                │
│                                                                     │
│  ┌─────────────────────────────────────────────┐                    │
│  │              Concat on Channel Dim            │                   │
│  │  [z_lq | x_hq_t] → [B, 8, H/8, W/8]        │                   │
│  └──────────────────┬──────────────────────────┘                    │
│                     │                                               │
│                     ↓                                               │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                    Encoder (3 stages)                        │    │
│  │                                                             │    │
│  │  Stage 1: ResidualSwinBlock                                 │    │
│  │    input [B,8,H/8,W/8]                                     │    │
│  │      ↓ Conv2d(8→128, 3×3)                                  │    │
│  │      ↓ LayerNorm → SwinTransformerBlock → Residual          │    │
│  │      ↓ LayerNorm → SwinTransformerBlock → Residual          │    │
│  │    output [B,128,H/8,W/8]                                  │    │
│  │                                                             │    │
│  │  Stage 2: ResidualSwinBlock                                 │    │
│  │    input [B,128,H/8,W/8]                                   │    │
│  │      ↓ Conv2d(128→256, 3×3, stride=2)                      │    │
│  │      ↓ SwinTransformerBlock ×2 + Residual                  │    │
│  │    output [B,256,H/16,W/16]                                │    │
│  │                                                             │    │
│  │  Stage 3: ResidualSwinBlock                                 │    │
│  │    input [B,256,H/16,W/16]                                 │    │
│  │      ↓ Conv2d(256→512, 3×3, stride=2)                      │    │
│  │      ↓ SwinTransformerBlock ×2 + Residual                  │    │
│  │    output [B,512,H/32,W/32]                                │    │
│  └──────────────────┬──────────────────────────────────────────┘    │
│                     │                                               │
│                     ↓                                               │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │              Transformer (2 ResidualAttentionBlocks)         │    │
│  │                                                             │    │
│  │  [B,512,H/32,W/32] → flatten → [B,L,512] (L=HW/1024)      │    │
│  │                                                             │    │
│  │  Block 1:                                                   │    │
│  │    ↓ LayerNorm → MultiHeadAttention(8 heads) → Residual     │    │
│  │    ↓ LayerNorm → Linear(512→2048) → GELU → Linear(2048→512) │    │
│  │    ↓ Residual                                               │    │
│  │                                                             │    │
│  │  Block 2: (same structure)                                  │    │
│  │    ↓ LayerNorm → MultiHeadAttention(8 heads) → Residual     │    │
│  │    ↓ LayerNorm → FFN → Residual                             │    │
│  │                                                             │    │
│  │  → reshape → [B,512,H/32,W/32]                             │    │
│  └──────────────────┬──────────────────────────────────────────┘    │
│                     │                                               │
│                     ↓                                               │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                    proj_out (zero_init)                      │    │
│  │                                                             │    │
│  │  [B,512,H/32,W/32]                                         │    │
│  │    ↓ Upsample ×4 (nearest + conv) → [B,512,H/8,W/8]        │    │
│  │    ↓ Conv2d(512→4, 3×3, padding=1, bias=True)               │    │
│  │    ↓ weight initialized to zero → output ≈ 0 at start       │    │
│  │  delta: [B,4,H/8,W/8]                                      │    │
│  └──────────────────┬──────────────────────────────────────────┘    │
│                     │                                               │
│                     ↓                                               │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                   Residual Connection                        │    │
│  │                                                             │    │
│  │  output = x_hq_t + delta                                   │    │
│  │                                                             │    │
│  │  训练初期: delta≈0 → output≈x_hq_t (不干扰UNet原始行为)      │    │
│  │  训练后期: delta 包含 LQ→HQ 的对齐修正信息                    │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  Output: aligned [B,4,H/8,W/8]                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### AlignmentModule 参数统计

| 子模块 | 参数量 | 说明 |
|--------|--------|------|
| Encoder Stage 1 (8→128) | ~0.5M | Conv + 2×SwinBlock |
| Encoder Stage 2 (128→256) | ~0.4M | Conv(stride=2) + 2×SwinBlock |
| Encoder Stage 3 (256→512) | ~0.4M | Conv(stride=2) + 2×SwinBlock |
| Transformer (2 blocks) | ~0.3M | 2×(SelfAttn + FFN) |
| proj_out (512→4) | ~0.01M | Upsample + Conv2d(zero_init) |
| **Total** | **~1.7M** | 仅占 UNet 总参数的 ~0.7% |

## 4. 注入方式对比：当前模型 vs FaithDiff

### 4.1 当前模型：替换式注入 (Replacement)

```
                    ┌─────────────────────────────────────────┐
                    │           UNet Forward                   │
                    │                                         │
  x_hq_t ──→ ┌─────────────┐                                │
              │ Alignment   │                                │
  z_lq ────→ │ Module      │──→ aligned                     │
              └─────────────┘       │                        │
                                    ↓                        │
                    ┌──────────────────────────────────────┐ │
                    │         conv_in (frozen)              │ │
                    │  conv2d(aligned) → conv_in_output     │ │
                    └──────────────────────────────────────┘ │
                                    │                        │
                                    ↓                        │
                    ┌──────────────────────────────────────┐ │
                    │   Down Blocks + Mid Block + Up Blocks │ │
                    │   (with LoRA, frozen)                 │ │
                    │   ← text_embed, timesteps             │ │
                    └──────────────────────────────────────┘ │
                                    │                        │
                                    ↓                        │
                              noise_pred                     │
                    └─────────────────────────────────────────┘

  注入位置: conv_in 之前（替换原始输入）
  注入方式: aligned 完全替代 x_hq_t 作为 conv_in 的输入
  信息空间: latent space (4ch, H/8×W/8)
```

### 4.2 FaithDiff：加法式注入 (Additive)

```
                    ┌─────────────────────────────────────────┐
                    │           UNet Forward                   │
                    │                                         │
                    │  sample (noisy latent)                   │
                    │       │                                 │
                    │       ↓                                 │
                    │  ┌──────────┐                           │
                    │  │ conv_in   │──→ input_embedding       │
                    │  └──────────┘         │                 │
                    │                       ↓                 │
                    │  ┌──────────────────────────────────┐   │
                    │  │ condition_embedding               │   │
                    │  │ (LQ pixel→VAE encode→transformer) │   │
                    │  │ → spatial_ch_proj → feat_alpha    │   │
                    │  └──────────────┬───────────────────┘   │
                    │                 │                       │
                    │                 ↓                       │
                    │  sample = input_embedding + feat_alpha  │
                    │                 │                       │
                    │                 ↓                       │
                    │  ┌──────────────────────────────────┐   │
                    │  │ Down Blocks + Mid + Up Blocks     │   │
                    │  │ (full params, trainable)          │   │
                    │  └──────────────────────────────────┘   │
                    │                 │                       │
                    │                 ↓                       │
                    │           noise_pred                    │
                    └─────────────────────────────────────────┘

  注入位置: conv_in 之后（在 embedding 空间叠加）
  注入方式: input_embedding + feat_alpha (加法残差)
  信息空间: conv_in 输出空间 (320ch, H/8×W/8)
```

### 4.3 关键差异总结

| 特性 | 当前模型 | FaithDiff |
|------|----------|-----------|
| **注入位置** | conv_in 之前 (4ch) | conv_in 之后 (320ch) |
| **注入方式** | 替换 (replacement) | 加法 (additive residual) |
| **信息空间** | latent (4ch) | embedding (320ch) |
| **信息容量** | 低 (4×64×64 = 16K/patch) | 高 (320×64×64 = 1.3M/patch) |
| **初始影响** | delta≈0 → 输出≈x_hq_t | feat_alpha≈0 → 不影响原始流程 |
| **LQ信息来源** | z_lq (latent space) | pixel-space encoded features |
| **UNet可训练性** | 全部冻结 (含LoRA) | Stage2 全部可训练 |
| **注入点数量** | 1 (conv_in) | 1 (conv_in后) 但信息量更大 |

## 6. FaithDiff 两阶段训练策略参考

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    FaithDiff Two-Stage Training                             │
│                                                                             │
│  Stage 1 (6000 steps, lr=5e-5):                                            │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  Trainable: condition_embedding + information_transformer +         │    │
│  │             spatial_ch_projs                                        │    │
│  │  Frozen:    UNet (all params), VAE, denoise_encoder                 │    │
│  │  Loss:      MSE(noise_pred, noise)                                  │    │
│  │  Purpose:   学习如何将 LQ 信息注入 UNet，不破坏预训练扩散先验          │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  Stage 2 (40000 steps, lr_UNet=5e-5, lr_encoder=5e-6):                    │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  Trainable: UNet (all params) + denoise_encoder                     │    │
│  │  Frozen:    condition_embedding + transformer + spatial_ch_projs     │    │
│  │             (加载 Stage 1 预训练权重并冻结)                           │    │
│  │  Loss:      MSE + perceptual + adversarial                          │    │
│  │  Purpose:   微调 UNet 利用注入的 LQ 信息，提升生成质量                │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  当前项目对应:                                                              │
│  Stage 1 ≈ 本项目的 alignment pretraining (训练 AlignmentModule)            │
│  Stage 2 ≈ 待实现 (冻结 alignment, 微调 UNet + LoRA)                       │
└─────────────────────────────────────────────────────────────────────────────┘
```
