# 训练步骤

## 1. 环境准备

```bash
conda activate hypir
```

确认依赖已安装：`torch`, `diffusers`, `accelerate`, `clip`, `einops`, `open_clip_torch`, `lpips`, `peft`, `omegaconf`

## 2. 数据准备

训练数据配置在 `configs/sd2_train.yaml` 中：

```yaml
file_list: data/LSDIR/lsdir_hypir.parquet   # 数据集索引文件
out_size: 512                                  # 训练 patch 大小
```

确保数据文件路径正确，parquet 文件包含 `image_path` 和 `prompt` 列。

## 3. 模型准备

需要预下载的模型：

| 模型 | 路径 | 说明 |
|------|------|------|
| Stable Diffusion 2.1 | `checkpoints/sd2` | 包含 vae、unet、text_encoder、tokenizer、scheduler |
| HYPIR 权重（微调时） | `checkpoints/HYPIR/HYPIR_sd2.pth` | 预训练 LoRA 权重，微调时加载作为起点 |

CLIP RN50（SeD 语义提取器）会在首次运行时自动下载。

## 4. 启动训练

```bash
cd SR-based-on-HYPIR

# 单卡训练
accelerate launch train.py --config configs/sd2_train.yaml

# 多卡训练（示例：2 卡）
accelerate launch --num_processes 2 train.py --config configs/sd2_train.yaml
```

## 5. 预训练权重加载

训练时会自动加载 `pretrained_weight_path` 指定的 HYPIR 预训练 LoRA 权重作为初始化。这比从头训练收敛更快、效果更好。

```yaml
# sd2_train.yaml 中已配置
pretrained_weight_path: checkpoints/HYPIR/HYPIR_sd2.pth
```

如果需要**从头训练**（不加载预训练权重），将该值设为 `null`：
```yaml
pretrained_weight_path: ~
```

## 6. 从 checkpoint 恢复训练

修改 `configs/sd2_train.yaml`：

```yaml
resume_from_checkpoint: output/checkpoint-1000
```

然后执行同样的训练命令。SeD 判别器的参数会用随机初始化（首次使用 SeD 时），原有 ConvNeXt D 的权重从 checkpoint 恢复。

## 7. 观察训练

### TensorBoard

```bash
tensorboard --logdir output/logs
```

### 关键 loss 指标

| 指标 | 含义 | 正常趋势 |
|------|------|----------|
| `loss/G_mse` | 像素 MSE 损失 | 持续下降 |
| `loss/G_lpips` | 感知损失 | 持续下降 |
| `loss/G_disc` | ConvNeXt D 对抗损失（生成器端） | 小幅波动 |
| `loss/G_sed` | SeD 语义对抗损失（生成器端） | 小幅波动 |
| `loss/D` | ConvNeXt D 判别器损失 | 稳定在 1-2 左右 |
| `loss/D_sed` | SeD 判别器损失 | 稳定在 1.4 左右（= 2×ln2） |
| `loss/D_logits_real` | ConvNeXt D 对真实图的 logits | 正值，逐渐稳定 |
| `loss/D_logits_fake` | ConvNeXt D 对生成图的 logits | 负值，逐渐稳定 |
| `loss/D_sed_logits_real` | SeD 对真实图的 logits | 正值 |
| `loss/D_sed_logits_fake` | SeD 对生成图的 logits | 负值 |

### 梯度监控

`loss/grad_norm/conv_out_l2`、`loss/grad_norm/conv_out_lpips`、`loss/grad_norm/conv_out_disc`、`loss/grad_norm/conv_out_sed`

如果 `sed` 梯度占比过高（>50%），考虑降低 `lambda_gan_sed`。

### 训练日志图像

每隔 `log_image_steps`（默认 100）步，保存在 `output/logs/log_images/` 下：

- `lq/` — 低质量输入
- `gt/` — 真实图像
- `G/` — 生成器输出
- `G_ema/` — EMA 模型输出（如果 use_ema=true）

## 8. 推理测试

```bash
bash test.sh
```

或手动执行：

```bash
python test.py \
  --base_model_type sd2 \
  --base_model_path checkpoints/sd2 \
  --model_t 200 \
  --coeff_t 200 \
  --lora_rank 256 \
  --lora_modules to_k,to_q,to_v,to_out.0,conv,conv1,conv2,conv_shortcut,conv_out,proj_in,proj_out,ff.net.2,ff.net.0.proj \
  --weight_path output/checkpoint-5000/state_dict.pth \
  --patch_size 512 \
  --stride 256 \
  --lq_dir examples/lq \
  --scale_by factor \
  --upscale 4 \
  --captioner empty \
  --output_dir results/test \
  --seed 231 \
  --device cuda
```

## 9. 关键配置说明

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| `max_train_steps` | 5000 | 总训练步数 |
| `batch_size` | 1 | 每卡 batch size |
| `lr_G` / `lr_D` | 1e-5 | 生成器/判别器学习率 |
| `lambda_gan` | 0.5 | ConvNeXt D 对抗损失权重 |
| `lambda_gan_sed` | 0.05 | SeD 语义对抗损失权重 |
| `lambda_lpips` | 5.0 | 感知损失权重 |
| `lambda_l2` | 1.0 | MSE 损失权重 |
| `gradient_checkpointing` | true | 节省显存 |
| `mixed_precision` | bf16 | 混合精度 |
| `checkpointing_steps` | 500 | 每 N 步保存 checkpoint |
| `checkpoints_total_limit` | 2 | 最多保留 N 个 checkpoint |
| `pretrained_weight_path` | `checkpoints/HYPIR/HYPIR_sd2.pth` | 预训练 HYPIR LoRA 权重路径，设为 null 则从头训练 |
