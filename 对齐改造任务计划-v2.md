# 对齐模块重新设计：完全参照 FaithDiff

## 状态：改造完成 ✅（2026-04-21）

## 问题诊断

当前对齐模块的两个根本性错误：

1. **UNet 接收 z_lq（LQ 的 latent），而非 x_hq_t（GT 加噪 latent）**
2. **没有加 t 步高斯噪声，使用 GAN loss 而非 L1 噪声预测 loss**

后果：对齐模块没有与 x_hq 对齐，只是冗余地将 RM 特征注入 UNet，不如直接把 x_en 喂给 UNet。

## FaithDiff 训练流程分析

### 核心流程（Stage 1 训练代码 `train_SDXL_stage_1.py:866-911`）

```python
# 1. GT → VAE encode → 加噪
hq_latents = vae.encode(GT) * scaling_factor
noise = torch.randn_like(hq_latents)
timesteps = torch.randint(0, T, (bsz,))          # 随机 timestep
noisy_latents = noise_scheduler.add_noise(hq_latents, noise, timesteps)  # x_hq_t

# 2. LQ → denoise_encoder → 对齐特征
fake_hq = unet.denoise_encoder(LQ)                # 像素空间编码

# 3. UNet 预测噪声（输入是 noisy GT，不是 LQ）
noise_pred = unet(noisy_latents, timesteps, text, input_embedding=fake_hq)

# 4. L1 噪声预测 loss
loss = L1(noise_pred, noise)
```

### 关键设计

| 要素 | FaithDiff | 当前错误实现 |
|------|-----------|-------------|
| UNet 输入 | `add_noise(encode(GT), noise, t)` = x_hq_t | `encode(LQ)` = z_lq |
| 对齐输入 | `denoise_encoder(LQ)` 像素空间 | `encode(RM(LQ))` latent 空间 |
| 时间步 | 随机 `randint(0, T)` | 固定 `t=200` |
| 加噪 | 有（标准扩散前向过程） | 无 |
| Loss | `L1(noise_pred, noise)` | GAN + L2 + LPIPS |
| 推理 | 多步 DDPM 去噪（20步） | 单步预测 |
| 判别器 | 不需要 | 需要（不必要） |

### 对齐的含义

"对齐"= 对齐模块从 LQ/RM(LQ) 提取特征，帮助 UNet 从 x_hq_t（加噪 GT）中恢复 x_hq。训练信号来自噪声预测 loss，对齐模块学习"如何让 LQ 特征帮助去噪 GT"。

## 新设计方案

### 整体架构

```
训练：
  GT → VAE.encode → z_hq → add_noise(t) → x_hq_t ──→ UNet ──→ noise_pred
                                              ↑              ↑
  LQ → RM → x_rm → VAE.encode → x_en → AlignmentHandler → features
                                              ↓
                                    Loss = L1(noise_pred, noise)

推理：
  x_en = VAE.encode(RM(LQ))
  alignment_features = AlignmentHandler(x_en)
  z = pure_noise
  for t in reversed(range(T)):
      noise_pred = UNet(z, t, text, alignment_features)
      z = scheduler.step(noise_pred, t, z).prev_sample
  HQ = VAE.decode(z)
```

### 与 FaithDiff 的差异

| | FaithDiff (SDXL) | 我们 (SD2) |
|---|---|---|
| 基础模型 | SDXL UNet | SD2 UNet + HYPIR LoRA |
| 对齐输入 | 像素空间 LQ (3ch) | latent 空间 RM(LQ) (4ch) |
| denoise_encoder | VAE encoder 初始化（冻结） | LatentAlignmentEncoder（可训练） |
| Stage 1 训练参数 | condition_embedding + transformer + proj | 同左 |
| Stage 2 训练参数 | 全部 UNet + denoise_encoder | LoRA + alignment |
| RM 预处理 | 无 | 有（DiffBIR RM，冻结） |

### 保留与删除

**保留**（架构正确，无需修改）：
- `alignment_module.py` — ControlNetConditioningEmbedding, ResidualAttentionBlock, zero_module
- `alignment_encoder.py` — LatentAlignmentEncoder（4ch latent → 512ch features，无空间下采样）
- `alignment_handler.py` — AlignmentHandler 整合流程（encode + fuse）
- `unet_alignment.py` — UNetAlignment wrapper

**重写**（训练流程错误）：
- `trainer/sd2_alignment.py` — prepare_batch_inputs, forward_generator, 移除 GAN
- `trainer/sd2_alignment_stage1.py` — 同上
- `enhancer/sd2_alignment.py` — 改为多步 DDPM 去噪推理
- `configs/*.yaml` — 更新训练配置
- `pipeline/rm_hypir_pipeline.py` — 适配新推理流程

## 实施阶段

### 阶段 1：重写训练流程

#### 任务 1.1：修改 `SD2AlignmentTrainer.prepare_batch_inputs()`

**当前（错误）**：
```python
z_lq = vae.encode(LQ)
x_en = vae.encode(RM(LQ))
timesteps = fixed(200)
```

**修改为**：
```python
# GT → latent → 加噪
z_hq = vae.encode(GT) * scaling_factor
noise = torch.randn_like(z_hq)
timesteps = torch.randint(0, num_train_timesteps, (bsz,))
noisy_hq = noise_scheduler.add_noise(z_hq, noise, timesteps)

# RM(LQ) → latent → 对齐输入
x_en = vae.encode(RM(LQ))

# 存入 batch_inputs
batch_inputs = {noisy_hq, noise, timesteps, x_en, text_embed, ...}
```

#### 任务 1.2：修改 `SD2AlignmentTrainer.forward_generator()`

**当前（错误）**：
```python
z_in = z_lq * scaling_factor
eps = G(z_in, t=200, text, x_en)
z = scheduler.step(eps, coeff_t, z_in).pred_original_sample
x = vae.decode(z / scaling_factor)
return x  # 用于 GAN + L2 + LPIPS loss
```

**修改为**：
```python
noise_pred = G(noisy_hq, timesteps, text, x_en=x_en).sample
return noise_pred  # 用于 L1(noise_pred, noise) loss
```

#### 任务 1.3：重写训练循环

- 移除 discriminator 相关代码（init_discriminator, optimize_discriminator, D loss）
- 训练循环简化为：
  ```python
  noise_pred = forward_generator()
  loss = F.l1_loss(noise_pred.float(), noise.float())
  loss.backward()
  optimizer.step()
  ```
- 不再需要 G/D 交替训练

#### 任务 1.4：修改 `SD2AlignmentStage1Trainer`

同上述修改，确保：
- Stage 1: 冻结 UNet+LoRA，只训练 alignment handler
- optimizer 只包含 alignment handler 参数

#### 任务 1.5：更新配置文件

**`configs/sd2_alignment_stage1_train.yaml`**：
```yaml
# 移除
```

**`configs/sd2_alignment_train.yaml`**：同上修改

### 阶段 2：重写推理流程

#### 任务 2.1：修改 `SD2AlignmentEnhancer`

**当前（错误）**：单步预测
```python
eps = G(z_lq, t=200, text, x_en=x_en)
z = scheduler.step(eps, 200, z_lq).pred_original_sample
x = vae.decode(z)
```

**修改为**：多步 DDPM 去噪
```python
# x_en 编码（与训练一致）
x_en = vae.encode(RM(LQ))

# 从纯噪声开始
z = torch.randn_like(x_en) * scheduler.init_noise_sigma

# 多步去噪
for t in scheduler.timesteps:
    noise_pred = G(z, t, text, x_en=x_en)
    z = scheduler.step(noise_pred, t, z).prev_sample

# VAE 解码
x = vae.decode(z / scaling_factor)
```

- 保留 RM 预处理步骤
