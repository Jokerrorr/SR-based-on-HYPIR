# 对齐模块改造计划 v3 — FaithDiff训练 + 单步推理

## 状态：进行中 🔄（2026-04-21）

## 背景

v2 版本将推理改为 20 步 DDPM 多步去噪，但这会严重拖慢推理速度（约20倍）。
HYPIR 的核心优势就是单步推理，需要保持这一特性。

## 已确认的设计决策

### 1. 时间步 t 固定为 200
- 与 HYPIR 原始设计一致
- 训练和推理的噪声水平一致
- 支持单步推理

### 2. Stage 1 可训练模块（与 FaithDiff 一致）
| 模块 | 状态 | 参数量 |
|------|------|--------|
| UNet | 冻结 | - |
| LoRA | 冻结 | - |
| alignment_encoder | 训练 | ~40M |
| condition_embedding | 训练 | ~0.5M |
| transformer_layers | 训练 | ~2M |
| spatial_ch_projs | 训练 | ~0.2M |

**总计约 42M 可训练参数**

## 方案：FaithDiff 训练 + 单步推理

### 核心思路

训练时使用 FaithDiff 的噪声预测训练（保证对齐模块学到正确的特征），
推理时使用 HYPIR 的单步模式（保持速度）。

关键：训练时的 t 固定为 `model_t=200`，推理时输入 `z_lq` 并假设它在 t=200 噪声水平，
UNet 一步预测噪声，scheduler 一步得到 `pred_original_sample`。

### 训练流程

```
GT → VAE.encode → z_hq → add_noise(noise, t=200) → noisy_hq ──→ UNet ──→ noise_pred
                                                          ↑              ↑
LQ → RM(冻结) → x_rm → VAE.encode → x_en → AlignmentHandler → features
                                                          ↓
                                                Loss = L1(noise_pred, noise)
```

- 时间步 t 固定为 200（匹配 HYPIR 原始设计）
- UNet 输入是 noisy_hq（GT 加噪），不是 z_lq
- 对齐特征来自 RM(LQ) 的 latent 编码

### 推理流程（单步）

```
z_lq = VAE.encode(LQ)
x_en = VAE.encode(RM(LQ))         # 对齐特征输入
noise = torch.randn_like(z_lq)
z_in = add_noise(z_lq, noise, t=200)  # 匹配训练时的噪声水平
alignment_features = AlignmentHandler(x_en)
noise_pred = UNet(z_in, t=200, text, alignment_features)
z_pred = scheduler.step(noise_pred, 200, z_in).pred_original_sample
HQ = VAE.decode(z_pred / scaling_factor)
```

- 只需 1 步 UNet forward，保持 HYPIR 的推理速度
- z_lq 加噪到 t=200 使输入噪声水平与训练一致

### 为什么训练用固定 t=200 而不是随机 t？

HYPIR 原始设计就是固定 t=200 单步推理。如果训练时用随机 t，
推理时固定 t=200 会造成训练/推理不一致。
固定 t=200 确保：
1. 训练和推理的噪声水平一致
2. 对齐模块在 t=200 这个特定噪声水平上优化
3. 与 HYPIR 原始设计兼容

## 详细修改方案

### 1. `HYPIR/enhancer/sd2_alignment.py`

当前是多步 DDPM 循环，改为单步：

```python
def enhance(self, lq, prompt, ...):
    # ... scale, pad 等前处理不变 ...

    # 编码对齐特征
    x_en = self.vae.encode(rm_norm).latent_dist.sample()

    # 编码 LQ latent
    z_lq = self.vae.encode(lq_norm).latent_dist.sample()

    # 加噪到 t=200（匹配训练）
    noise = torch.randn_like(z_lq)
    model_t = 200
    z_in = self.scheduler.add_noise(z_lq, noise, torch.tensor([model_t]))

    # 单步预测
    self.prepare_inputs(batch_size=bs, prompt=prompt)
    noise_pred = self.G(
        z_in, torch.tensor([model_t], device=self.device),
        encoder_hidden_states=self.inputs["c_txt"]["text_embed"],
        x_en=x_en,
    ).sample

    # 一步得到预测
    z_pred = self.scheduler.step(noise_pred, model_t, z_in).pred_original_sample

    # VAE 解码
    z_pred = z_pred / self.vae.config.scaling_factor
    x = self.vae.decode(z_pred.to(self.weight_dtype)).sample.float()

    # ... 后处理不变 ...
```

### 2. `HYPIR/trainer/sd2_alignment.py`

`prepare_batch_inputs()` 中将随机 t 改为固定 t=200：

```python
# 当前（v2）
timesteps = torch.randint(0, self.noise_scheduler.num_train_timesteps, (bsz,), ...)

# 改为
model_t = getattr(self.config, "model_t", 200)
timesteps = torch.full((bsz,), model_t, dtype=torch.long, device=self.device)
```

**训练流程**：
- `prepare_batch_inputs()`: 使用固定 `model_t=200` 而非随机 timestep
- GT → VAE encode → add_noise(t=200) → noisy_hq
- RM(LQ) → VAE encode → x_en (对齐特征输入)
- Loss = L1(noise_pred, noise)

- z_lq = VAE.encode(LQ)

- noise_pred = UNet(z_in, t=200, text, alignment_features)
