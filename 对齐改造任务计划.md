# 对齐改造任务计划

## 整体目标

改造FaithDiff的对齐模块，插入到HYPIR的VAE Encoder与UNet之间：

```
LQ → RM(DiffBIR) → VAE Encoder(HYPIR) → x_en → Alignment(FaithDiff改造) → UNet(HYPIR) → VAE Decoder(HYPIR) → HQ
```

**关键设计**：对齐模块以 **x_en（VAE Encoder对RM输出的编码，latent space）** 与 **x_hq** 对齐。对齐模块工作在VAE latent空间（4通道，H/8 × W/8），不需要空间下采样。

- `init_rm()`：根据配置加载冻结的RM模型（SCUNet for BID）

## 阶段7：二阶段训练设计（当前阶段）

### 背景与动机

FaithDiff采用两阶段训练策略，已被验证有效。我们借鉴该思路，将当前的单阶段对抗训练拆分为两个阶段，逐步优化对齐模块和UNet。

#### FaithDiff两阶段训练分析

**Stage 1（对齐预训练）** — `train_SDXL_stage_1.py`：
- **目标**：训练对齐模块（denoise_encoder + condition_embedding + transformer + spatial_ch_projs）
- **训练方式**：标准扩散噪声预测训练
- **冻结**：UNet backbone全部冻结（`unet.train()`但只优化alignment相关参数）
- **优化参数**：仅 `condition_embedding + information_transformer_layes + spatial_ch_projs`（denoise_encoder用VAE encoder初始化也冻结）
- **Loss**：`L1(noise_pred, noise)` — 纯噪声预测loss
- **数据流**：
  ```
  hq_latents = VAE.encode(GT) * scaling_factor
  t = random(0, T)
  noisy_latents = add_noise(hq_latents, noise, t)    # x_hq_t
  input_emb = denoise_encoder(LQ)                      # 对齐编码（冻结）
  eps_pred = UNet(noisy_latents, t, text, input_emb)   # UNet冻结，只训练alignment注入
  loss = L1(eps_pred, noise)
  ```
- **输出**：`pretrain.bin`（Stage 1训练好的alignment模块权重）

**Stage 2（联合微调）** — `train_SDXL_stage_2.py`：
- **目标**：联合训练对齐模块和UNet backbone
- **初始化**：加载Stage 1的`pretrain.bin`权重
- **训练方式**：仍然使用扩散噪声预测训练（与Stage 1相同的loss）
- **解冻**：UNet backbone解冻，全部参数可训练
- **优化参数**：denoise_encoder（lr=5e-5）+ 其余UNet参数（lr=1e-4），两组学习率
- **Loss**：`L1(noise_pred, noise)` — 仍然是噪声预测loss
- **验证**：使用完整pipeline推理（20步DDPM去噪），计算MUSIQ质量指标

#### 我们的二阶段训练设计

与FaithDiff的关键差异：
- FaithDiff使用**标准扩散训练**（随机t，噪声预测loss）
- 我们使用**HYPIR的对抗训练框架**（固定t=200，GAN + L2 + LPIPS loss）
- FaithDiff的denoise_encoder工作在**像素空间**（3ch），我们的AlignmentEncoder工作在**latent空间**（4ch）

**我们的Stage 1（对齐预训练）**：
- 借鉴FaithDiff Stage 1思路：**冻结UNet backbone，只训练对齐模块**
- 但使用HYPIR的对抗训练框架
- 目的：让对齐模块学会从x_en提取有用特征，稳定初始化

**我们的Stage 2（联合对抗训练）**：
- 借鉴FaithDiff Stage 2思路：**解冻UNet LoRA，联合训练对齐模块+LoRA**
- 与当前`SD2AlignmentTrainer`基本一致，但从Stage 1 checkpoint初始化

---

### 任务7.1：Stage 1 对齐预训练Trainer — `HYPIR/trainer/sd2_alignment_stage1.py`

**核心设计**：冻结UNet LoRA，只训练Alignment Handler参数

- 继承`SD2AlignmentTrainer`，覆写以下方法：
  - `init_generator()`：加载UNet + Alignment Handler，但**不添加LoRA**，UNet全部冻结
  - `init_discriminator()`：仍然初始化Discriminator
  - `init_optimizers()`：只包含alignment handler参数
  - `forward_generator()`：同父类（使用x_en）
  - `optimize_generator()`：同父类（GAN loss + L2 + LPIPS）
  - `optimize_discriminator()`：同父类
- **不需要**文本编码器参与（UNet冻结后text conditioning仍通过cross attention）
- 保存：仅alignment handler权重

**配置要点**：
```yaml
# Stage 1: 只训练alignment，UNet完全冻结
base_model_type: sd2_alignment_stage1
use_lora: false                # 不使用LoRA
freeze_unet: true              # UNet完全冻结
alignment_lr: 1e-4             # alignment专用学习率
max_train_steps: 3000          # 预训练步数较少
```

### 任务7.3：Stage 2 联合训练Trainer改造 — 修改`SD2AlignmentTrainer`

**核心变更**：支持从Stage 1 checkpoint加载alignment权重

- 新增配置项：`alignment_pretrained_path`（Stage 1输出路径）
- `init_generator()`中：
  1. 创建UNet + LoRA + Alignment Handler（与当前相同）
  2. 如果`alignment_pretrained_path`存在，加载Stage 1训练好的alignment权重
  3. 对齐模块仍可训练（不冻结）
- 其余训练逻辑不变

**配置要点**：
```yaml
# Stage 2: 联合训练 LoRA + Alignment
base_model_type: sd2_alignment
alignment_pretrained_path: output_alignment_stage1/checkpoint-xxx/state_dict.pth
# 其余配置同当前 sd2_alignment_train.yaml
```
