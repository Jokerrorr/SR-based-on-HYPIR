## 当前状态：阶段8进行中（推理测试管道补全）

### 架构修正记录

**修正内容1**：对齐模块的输入从像素空间x_rm改为VAE latent空间x_en。
**修正原因**：对齐模块应当以VAE ENCODER的输出x_en与x_hq对齐，而非直接处理像素空间的RM输出。
**影响范围**：所有代码文件和文档已全部修正。

**修正内容2**：两阶段训练均加载HYPIR官方预训练LoRA权重作为初始化。
**修正原因**：原设计从SD2基础模型开始训练，忽略了HYPIR已有的图像恢复能力（514个LoRA参数），导致收敛慢且浪费预训练成果。
**影响范围**：Stage 1/2 Trainer的init_generator()、配置文件、训练流程文档。

### 数据流（修正后）

```
LQ → RM → x_rm(像素) → VAE Encoder → x_en(latent, 4ch) → Alignment → UNet → VAE Decoder → HQ
```

对齐模块完全在latent space工作（4ch, H/8×W/8），无空间下采样。

## 当前阶段：阶段7 — 二阶段训练设计 🔄

### 设计思路

借鉴FaithDiff的两阶段训练策略，**两个阶段均以HYPIR官方预训练LoRA为起点**：

完整数据流：
```
LQ → RM(DiffBIR,冻结) → VAE Encoder(HYPIR,冻结) → x_en → Alignment → UNet(HYPIR+LoRA) → VAE Decoder(HYPIR,冻结) → HQ
```

- **Stage 1**：冻结RM、VAE、UNet+LoRA(HYPIR预训练)，只训练Alignment Handler
- **Stage 2**：冻结RM、VAE，解冻UNet LoRA(HYPIR预训练) + Alignment Handler(Stage1预训练)，联合微调

### HYPIR预训练权重

| 文件 | 来源 | 路径 |
|------|------|------|
| `HYPIR_sd2.pth` | 官方预训练 | `checkpoints/HYPIR/HYPIR_sd2.pth` |
| `state_dict-LSDIR50-5000.pth` | 用户自行训练 | `checkpoints/HYPIR/state_dict-LSDIR50-5000.pth` |

两个文件key结构完全相同（514个LoRA参数），使用官方预训练作为训练初始化。

### 与FaithDiff的差异

| | FaithDiff Stage 1 | 我们的 Stage 1 |
|---|---|---|
| UNet状态 | 冻结 | 冻结 |
| LoRA | 无 | 无 |
| 训练参数 | alignment模块 | alignment模块 |
| Loss | L1噪声预测 | GAN + L2 + LPIPS |
| 时间步t | 随机 | 固定200 |
| 输入空间 | 像素(3ch) | latent(4ch) |

| | FaithDiff Stage 2 | 我们的 Stage 2 |
|---|---|---|
| UNet状态 | 全部解冻 | LoRA解冻 |
| 训练参数 | UNet全部 + alignment | LoRA + alignment |
| Loss | L1噪声预测 | GAN + L2 + LPIPS |
| 多学习率 | denoise_encoder/UNet分离 | alignment/LoRA可分离 |

**8.1 SD2AlignmentEnhancer.init_generator改造**：
- 原实现：先创建UNet+LoRA → 加载LoRA权重到unet → 创建handler → 包装

- `UNetAlignment.load_state_dict(state_dict, strict=False)`自动分离`unet.*`和`alignment_handler.*`键
- 支持两种权重文件：HYPIR原始LoRA（514键）和二阶段训练权重（LoRA+alignment）

**8.2 test.py对齐模式**：

- 默认使用HYPIR原始LoRA做冒烟测试

### 3. Alignment Encoder初始化 — ✅ 已确认
