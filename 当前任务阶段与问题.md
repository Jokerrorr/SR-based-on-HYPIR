## 当前状态：v4 Stage 2 对抗微调改造 ✅

### v4 核心改动

1. **对齐模块输入**：从单侧 x_en 改为双侧 (z_lq, x_hq_t)
2. **对齐模块输出**：从 320ch 改为 4ch（直接作为 UNet 输入）
3. **注入位置**：从 conv_in 之后移到 conv_in 之前
4. **Stage 1 训练**：L1 噪声预测损失（冻结 UNet+LoRA，只训 alignment）
5. **Stage 2 训练**：HYPIR 对抗损失 + 对齐 L1 损失（G/D 交替）

### 数据流

**训练 Stage 1（对齐预训练）**：
```
GT → VAE.encode → z_hq → add_noise(t=200) → x_hq_t ─┐
                                                       ├→ AlignmentModule → aligned → UNet → noise_pred
RM(LQ) → VAE.encode → z_lq ──────────────────────────┘
Loss = L1(noise_pred, noise)
```

**训练 Stage 2（对抗微调）**：
```
GT → VAE.encode → z_hq → add_noise(t=200) → x_hq_t ─┐
                                                       ├→ AlignmentModule → aligned → UNet → noise_pred
RM(LQ) → VAE.encode → z_lq ──────────────────────────┘
                                                       ↓
                                              scheduler.step → z_pred → VAE.decode → x_pred
Loss_G = λ_l2·MSE(x_pred, gt) + λ_lpips·LPIPS(x_pred, gt) + λ_gan·D(x_pred) + λ_align·L1(noise_pred, noise)
Loss_D = D(gt, real) + D(x_pred, fake)
G/D 交替训练
```

**推理（单步）**：
```
RM(LQ) → VAE.encode → z_lq ─┐
                               ├→ AlignmentModule → aligned → UNet(t=200) → z_pred → VAE.decode → HQ
z_lq + noise(t=200) ─────────┘  (近似 x_hq_t)
```

### 两阶段训练策略

| | Stage 1 | Stage 2 |
|--|---------|---------|
| UNet+LoRA | 冻结（加载 HYPIR 预训练） | 可训练（加载 HYPIR 预训练） |
| AlignmentHandler | 可训练（从零开始） | 可训练（加载 Stage 1 权重） |
| 损失 | L1(noise_pred, noise) | λ_l2·MSE + λ_lpips·LPIPS + λ_gan·D + λ_align·L1 |
| G/D 交替 | 无 | 有 |
| 输出空间 | latent（noise_pred） | pixel（x_pred via VAE decode） |
| 学习率 | alignment_lr=1e-4 | lr_G=1e-5, lr_D=1e-5 |

### 损失权重

| 损失项 | 权重 | 说明 |
|--------|------|------|
| lambda_l2 | 1.0 | 像素空间 MSE |
| lambda_lpips | 5.0 | 感知损失 |
| lambda_gan | 0.5 | 对抗损失 |
| lambda_align | 1.0 | 对齐噪声预测 L1 |

### 对齐模块结构

```python
class AlignmentModule:
    forward(z_lq, x_hq_t):
        x = concat(z_lq, x_hq_t)  # [B, 8, H, W]
        x = encoder(x)             # [B, 512, H, W]
        x = transformer(x)         # [B, 512, H, W]
        delta = proj_out(x)        # [B, 4, H, W] (zero_init)
        return x_hq_t + delta      # 残差，初始≈x_hq_t
```

## 已修复的问题 ✅

### 1. Dtype 不匹配（已修复）

**问题**：`RuntimeError: Input type (c10::BFloat16) and bias type (float) should be the same`

**原因**：Alignment handler 参数为 fp32（训练稳定性），但 VAE 输出为 bf16

**修复**：在 `unet_alignment.py` 中添加 dtype 转换：
```python
handler_dtype = next(self.alignment_handler.parameters()).dtype
aligned = self.alignment_handler(
    z_lq.to(dtype=handler_dtype),  # bf16 → fp32
    x_hq_t.to(dtype=handler_dtype)
)
unet_dtype = self.unet.conv_in.weight.dtype
aligned = aligned.to(dtype=unet_dtype)  # fp32 → bf16
```

### 2. 梯度流被阻断（已修复）

**问题**：`torch.no_grad()` 包裹 conv_in 调用导致梯度无法回传到 alignment handler

**修复**：移除 `torch.no_grad()` 包装，梯度可正常流经 conv_in 到 alignment handler

### 3. zero_init 梯度传播行为（已验证正常）

**现象**：初始化时只有 proj_out (2/34) 参数接收梯度

**原因**：这是 zero_init 的预期行为：
- proj_out 初始化为零，输出 `aligned ≈ x_hq_t`
- 梯度只影响 proj_out（其他层输出被零乘）
- 一次 optimizer.step 后，proj_out 权重非零，梯度传播到所有层

**验证**：测试确认一次训练步后 34/34 参数均接收梯度
