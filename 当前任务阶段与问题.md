## 当前状态：FaithDiff-style 加法注入改造

### 核心改动（相对 v4）

1. **注入方式**：从 conv_in 前替换式 → conv_in 后加法残差
2. **对齐模块**：从 SwinBlock encoder → FaithDiff-style condition_embedding + transformer
3. **输出通道**：从 4ch → 320ch（匹配 UNet 内部特征空间）
4. **信息容量**：从 16K/patch → 1.3M/patch（80倍提升）

### 数据流

**训练 Stage 1**：
```
GT → VAE.encode → z_hq → add_noise(t=200) → x_hq_t → conv_in → sample_emb ─┐
                                                                              ├→ sample_emb + feat_alpha → UNet → noise_pred
RM(LQ) → VAE.encode → z_lq → FaithDiffAlignment(sample_emb, z_lq) ─────────┘
Loss = L1(noise_pred, noise)
```

**推理（单步）**：
```
RM(LQ) → VAE.encode → z_lq
z_lq + noise(t=200) → conv_in → sample_emb
sample_emb + FaithDiffAlignment(sample_emb, z_lq) → UNet → noise_pred
→ scheduler.step → z_pred → VAE.decode → HQ
```

### 对齐模块结构（FaithDiff-style）

```python
class FaithDiffAlignment:
    forward(sample_emb, z_lq):
        input_embedding = condition_embedding(z_lq)        # [B,4] → [B,320]
        concat_feat = cat([sample_emb, input_embedding])   # [B,640]
        concat_feat = transformer(concat_feat)              # [B,640]
        feat_alpha = spatial_ch_proj(concat_feat)           # [B,320] (zero_init)
        return feat_alpha
```

注入：`sample_emb = sample_emb + feat_alpha`

### 与 FaithDiff 原始实现对比

| 组件 | FaithDiff 原始 | 当前实现 |
|------|---------------|---------|
| LQ 编码 | denoise_encoder (独立VAE) | RM + VAE.encode |
| condition_embedding | ControlNetConditioningEmbedding | 照搬 |
| information_transformer | ResidualAttentionBlock × 2 | 照搬 |
| spatial_ch_proj | Linear(640→320, zero_init) | 照搬 |
| 注入方式 | `sample + feat_alpha` | 照搬 |

## 改造问题分析（v4 → 当前）

### v4 存在的问题

| 问题 | 说明 | 当前修复 |
|------|------|---------|
| **信息瓶颈** | 512→4ch proj_out 压缩 128 倍 | 320ch 直接注入 |
| **注入位置不当** | conv_in 前特征被预训练权重过滤 | conv_in 后直接注入 |
| **替换式注入** | 破坏预训练 UNet 输入分布 | 加法残差，初始为零 |
| **信息容量不足** | 16K/patch | 1.3M/patch (80x) |
