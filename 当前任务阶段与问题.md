## 当前状态：阶段7已完成（二阶段训练设计+验证）

### 架构修正记录

**修正内容**：对齐模块的输入从像素空间x_rm改为VAE latent空间x_en。
**修正原因**：对齐模块应当以VAE ENCODER的输出x_en与x_hq对齐，而非直接处理像素空间的RM输出。
**影响范围**：所有代码文件和文档已全部修正。

### 数据流（修正后）

```
LQ → RM → x_rm(像素) → VAE Encoder → x_en(latent, 4ch) → Alignment → UNet → VAE Decoder → HQ
```

对齐模块完全在latent space工作（4ch, H/8×W/8），无空间下采样。

## 当前阶段：阶段7 — 二阶段训练设计 🔄

### 设计思路

借鉴FaithDiff的两阶段训练策略：
- **Stage 1**：冻结UNet backbone，只训练Alignment Handler（预训练对齐模块）
- **Stage 2**：解冻UNet LoRA，联合训练对齐模块+LoRA（从Stage 1权重初始化）

### 与FaithDiff的差异

| | FaithDiff Stage 1 | 我们的 Stage 1 |
|---|---|---|
| UNet状态 | 冻结 | 冻结 |
| LoRA | 无 | 无 |
| 训练参数 | alignment模块 | alignment模块 |
| Loss | L1噪声预测 | GAN + L2 + LPIPS |
| 时间步t | 随机 | 固定200 |
| 输入空间 | 像素(3ch) | latent(4ch) |

| | FaithDiff Stage 2 | 我们的 Stage 2 |
|---|---|---|
| UNet状态 | 全部解冻 | LoRA解冻 |
| 训练参数 | UNet全部 + alignment | LoRA + alignment |
| Loss | L1噪声预测 | GAN + L2 + LPIPS |
| 多学习率 | denoise_encoder/UNet分离 | alignment/LoRA可分离 |
