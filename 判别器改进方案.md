# 判别器改进方案

## 1. 改进目标

在 HYPIR 训练中新增 SeD（Semantic-aware Discriminator）作为**补充判别器**，为原有 ConvNeXt 判别器增加语义一致性引导能力。

核心思路：**不替换、只新增**。原有 ConvNeXt D 在 512×512 全分辨率上提供细节判别，SeD 在 256×256 分辨率上提供语义一致性判别。

## 2. 改进前后对比

### 改进前（原始 HYPIR）

```
L_G = 1.0 × L_mse + 5.0 × L_lpips + 0.5 × L_adv_convnext
```

只有 ConvNeXt D 提供对抗信号，无语义引导。

### 改进后（新增 SeD）

```
L_G = 1.0 × L_mse + 5.0 × L_lpips + 0.5 × L_adv_convnext + 0.05 × L_adv_sed
                                            ↑ 保留不变          ↑ 新增
```

## 3. 损失系数设置

| 损失项 | 系数 | 损失类型 | 来源 | 说明 |
|--------|------|----------|------|------|
| L_mse | 1.0 | MSE | HYPIR 原始 | 不变 |
| L_lpips | 5.0 | LPIPS | HYPIR 原始 | 不变 |
| L_adv_convnext | 0.5 | Multilevel Hinge | HYPIR 原始 D | 不变 |
| L_adv_sed | **0.05** | BCEWithLogits | **新增** SeD_P | SeD 源码原始 loss_g=0.01，适配 HYPIR scale 后为 0.05 |

### 系数推导过程

1. **SeD 源码参考**：`basecode/SeD/options/train_rrdb_P+SeD.yml` 中 `loss_g: 0.01`
2. **HYPIR 原始 scale**：`lambda_gan: 0.5`（50 倍于 SeD 原始设置）
3. **推导**：HYPIR 的 L_mse=1.0, L_lpips=5.0，整体 loss scale 远大于 SeD 原始环境（SeD 原始 loss_pix=1.0, loss_p=0.1），因此 SeD GAN loss 需要适当放大
4. **最终选择 0.05**：
   - 约为 ConvNeXt D 权重 0.5 的 1/10 → SeD 扮演语义正则化器
   - 总对抗权重 0.55，增幅仅 10% → 训练稳定性风险低
   - 后续可逐步调大（0.05 → 0.1 → 0.2）

## 4. 文件变更明细

### 4.1 新增文件

#### `HYPIR/model/module_attention.py` — SeFB 交叉注意力模块

来源：`basecode/SeD/models/module_attention.py`

包含类：
- `checkpoint()` / `CheckpointFunction` — 梯度检查点
- `GEGLU` / `FeedForward` — 前馈网络
- `CrossAttention` — 交叉注意力（Q=语义, K/V=图像）
- `BasicTransformerBlock` — SA→CrossAttn→FF（无残差连接）
- `ModifiedSpatialTransformer` — SeFB 主体（GroupNorm→proj→transformer→pixel shuffle→conv）

移除内容：`SpatialSelfAttention`、`LinearAttention`（未使用）

#### `HYPIR/model/sed_discriminator.py` — 语义提取器 + SeD_P 判别器

来源：`basecode/SeD/models/sed.py`

包含类：
- `CLIP_Semantic_extractor` — 继承 `clip.model.ModifiedResNet`，冻结，输出 (B,1024,16,16)
- `SeD_P` — PatchGAN 判别器，3 个 SeFB 注入点，输出 (B,1,30,30)

### 4.2 修改文件

#### `HYPIR/trainer/base.py` — 训练器核心

需修改的方法：

| 方法 | 修改内容 |
|------|----------|
| 顶部 import | 新增 `from HYPIR.model.sed_discriminator import CLIP_Semantic_extractor, SeD_P` |
| `init_models()` | 新增 `self.init_sed_discriminator()` 调用 |
| `init_sed_discriminator()` | **新增方法**：创建 semantic_extractor + SeD_P |
| `_prepare_sed_inputs()` | **新增方法**：resize 到 256 + 提取 GT 语义特征 |
| `init_optimizers()` | 新增 D_sed_params + D_sed_opt |
| `prepare_all()` | attrs 扩展为 `["G", "D", "D_sed", "G_opt", "D_opt", "D_sed_opt", "dataloader"]` |
| `optimize_generator()` | D_sed 设为 eval；新增 loss_sed 加入 loss_G；loss_dict 增加 G_sed |
| `optimize_discriminator()` | 新增 SeD D 训练块（BCE real/fake）；loss_dict 增加 D_sed 相关键 |
| `log_grads()` | D_sed 设为 eval；losses 列表增加 ("sed", loss_sed) |

#### `configs/sd2_train.yaml` — 训练配置

新增配置项：
```yaml
lambda_gan_sed: 0.05    # SeD GAN loss 权重
sed_type: "P"            # P=PatchGAN
sed_resize: 256          # resize 目标尺寸
```

### 4.3 不修改的文件

| 文件 | 原因 |
|------|------|
| `HYPIR/model/D.py` | 原始 ImageConvNextDiscriminator 保留不动 |
| `HYPIR/model/backbone.py` | ConvNeXt 骨干保留不动 |
| `HYPIR/trainer/sd2.py` | 判别器逻辑全在 base.py |
| `train.py` | 入口文件无需修改 |
| `requirements.txt` | clip、einops 已安装 |

## 8. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| resize 导致细节判别丢失 | ConvNeXt D 在全分辨率补偿 |
| SeD vanilla GAN 不稳定 | 监控 D_sed logits，必要时换 hinge loss |
| 双 D 梯度冲突 | SeD 权重小(0.05)作为正则化器 |
| resume checkpoint 不兼容 | SeD 新增参数随机初始化 |
