# 对齐模块改造计划 v4 — FaithDiff式双侧对齐

## 状态：代码完成 ✅（2026-04-22）

---

## 背景

### v1~v3 的问题

v1~v3 的对齐模块设计偏离了 FaithDiff 的核心思想：

**当前实现（错误）**：
```
GT → VAE Encoder → z_hq → add_noise → noisy_hq → UNet conv_in → conv_out ─┐
                                                                             ├→ fuse → 替换conv_in输出
RM(LQ) → VAE Encoder → x_en → LatentAlignmentEncoder → features ───────────┘
```

这是 **ControlNet 式的单侧注入**，不是 FaithDiff 的对齐方式。

**FaithDiff 正确设计**：
```
GT → VAE Encoder(冻结) → z_hq → add_noise(t) → x_hq_t ─┐
                                                          ├→ 对齐模块 → aligned → UNet输入
LQ → LQ Encoder(可训练) → x_lq ─────────────────────────┘
```

对齐模块的输入是**两个**：x_lq 和 x_hq_t，负责把它们对齐，输出才是 UNet 的输入。

---

## v4 设计方案

### 核心思路

使用**同一个 VAE Encoder** 编码两边：

```
GT → VAE Encoder(冻结) → z_hq → add_noise(t=200) → x_hq_t ─┐
                                                               ├→ 对齐模块 → aligned → UNet
RM(LQ) → VAE Encoder(冻结) → z_lq ───────────────────────────┘
```

**好处**：
1. z_lq 和 x_hq_t 已经在同一个潜空间（4ch, H/8），对齐任务更简单
2. 不需要额外的像素空间 LQ Encoder
3. RM 已经修复了大部分退化，VAE.encode(RM(LQ)) 质量有保障

### 对齐模块设计

```
z_lq [4ch] ──┐
              ├→ concat → [8ch] → 扩展处理 → Transformer对齐 → 投影 → [4ch]
x_hq_t [4ch] ─┘
```

```python
class AlignmentModule(nn.Module):
    """FaithDiff式双侧对齐模块"""

    def __init__(self, latent_channels=4, hidden_channels=128):
        # 8ch → 128 → 256 → 512 中间扩展
        self.encoder = ...  # concat(z_lq, x_hq_t) → high-dim features

        # Transformer 对齐
        self.transformer = ...  # 在高维空间做注意力对齐

        # 512 → 4ch 投影，zero_init 保证初始输出≈0
        self.proj_out = zero_module(nn.Conv2d(512, latent_channels, 1))

    def forward(self, z_lq, x_hq_t):
        x = concat(z_lq, x_hq_t)  # [B, 8, H/8, W/8]
        x = self.encoder(x)        # [B, 512, H/8, W/8]
        x = self.transformer(x)    # [B, 512, H/8, W/8]
        x = self.proj_out(x)       # [B, 4, H/8, W/8]
        return x_hq_t + x          # 残差，初始时≈x_hq_t
```

**关键点**：`zero_init` 的 `proj_out` 保证训练初期 `aligned ≈ x_hq_t`，模型逐步学会从 z_lq 提取有用信息做修正。

### 训练流程

**Stage 1**（对齐预训练）：
```
GT → VAE.encode → z_hq → add_noise(t=200) → x_hq_t ─┐
                                                       ├→ AlignmentModule → aligned → UNet → noise_pred
RM(LQ) → VAE.encode → z_lq ──────────────────────────┘

Loss = L1(noise_pred, noise)
```

冻结：RM、VAE、UNet+LoRA(HYPIR预训练)
训练：AlignmentModule

**Stage 2**（对抗微调）：
在 HYPIR 原始对抗训练基础上，增加对齐 L1 噪声预测损失。

```
GT → VAE.encode → z_hq → add_noise(t=200) → x_hq_t ─┐
                                                       ├→ AlignmentModule → aligned → UNet → noise_pred
RM(LQ) → VAE.encode → z_lq ──────────────────────────┘
                                                       ↓
                                              scheduler.step → z_pred → VAE.decode → x_pred

Loss_G = λ_l2·MSE(x_pred, gt) + λ_lpips·LPIPS(x_pred, gt) + λ_gan·D(x_pred) + λ_align·L1(noise_pred, noise)
Loss_D = D(gt, real) + D(x_pred, fake)
G/D 交替训练
```

冻结：RM、VAE
训练：LoRA + AlignmentModule（加载 Stage 1 权重联合训练）

### 推理流程

```
LQ → RM → VAE.encode → z_lq ─┐
                                ├→ AlignmentModule → aligned → UNet(t=200) → z_pred → VAE.decode → HQ
LQ → VAE.encode → z_lq' ──────┘  (加噪到t=200作为x_hq_t)
```

推理时没有 GT，用 LQ 的 VAE 编码加噪作为 x_hq_t 近似。

## 实施步骤

### Phase 1：对齐模块重构（P0）

1. **新增 AlignmentModule**
   - 输入：z_lq [B,4,H,W], x_hq_t [B,4,H,W]
   - 输出：aligned [B,4,H,W]
   - 结构：concat → encoder → transformer → proj_out → 残差

2. **重写 AlignmentHandler**
   - 移除 LatentAlignmentEncoder（不再需要4→512扩展）
   - 改为调用 AlignmentModule(z_lq, x_hq_t)

3. **修改 UNetAlignment**
   - 注入点从 conv_in 之后移到 conv_in 之前
   - forward 接收 z_lq 和 x_hq_t，先对齐再进 conv_in

### Phase 2：训练器改造（P0）

1. **prepare_batch_inputs()**
   ```python
   # GT → VAE → add_noise → x_hq_t
   z_hq = self.vae.encode(gt)
   x_hq_t = self.scheduler.add_noise(z_hq, noise, t=200)

   # RM(LQ) → VAE → z_lq
   x_rm = self.rm.inference(lq)
   z_lq = self.vae.encode(x_rm)
   ```

2. **forward_generator()**
   ```python
   # Stage 1: 返回 noise_pred (latent space)
   noise_pred = self.G(x_hq_t, t=200, text_embed, z_lq=z_lq, x_hq_t=x_hq_t)
   return noise_pred

   # Stage 2: 返回 x_pred (pixel space)
   noise_pred = self.G(x_hq_t, t=200, text_embed, z_lq=z_lq, x_hq_t=x_hq_t)
   z_pred = self.scheduler.step(noise_pred, t=200, x_hq_t)
   x_pred = self.vae.decode(z_pred)
   return x_pred
   ```

3. **optimize_generator()** (Stage 2)
   ```python
   x_pred = self.forward_generator()
   loss_l2 = MSE(x_pred, gt) * lambda_l2
   loss_lpips = LPIPS(x_pred, gt) * lambda_lpips
   loss_disc = D(x_pred, for_G=True) * lambda_gan
   loss_align = L1(noise_pred, noise) * lambda_align  # 使用缓存的 noise_pred
   loss_G = loss_l2 + loss_lpips + loss_disc + loss_align
   ```

## 风险与待定问题

### 1. 推理时 x_hq_t 近似

推理时用 `add_noise(VAE.encode(LQ), t=200)` 近似 x_hq_t。这假设 LQ 和 GT 在 VAE 潜空间接近，对于严重退化可能不准。

**缓解**：RM 已经修复了大部分退化，z_lq = VAE.encode(RM(LQ)) 质量较好。

### 2. 是否需要像素空间 LQ Encoder

当前设计用同一个 VAE 编码两边。如果效果不好，可考虑：
- 添加独立的像素空间 LQ Encoder（类似 FaithDiff 原版）
- 或用 ControlNet encoder 结构

### 3. 训练损失

**Stage 1**：L1 噪声预测损失（FaithDiff 方式），冻结 UNet+LoRA，只训练 alignment。

**Stage 2**：HYPIR 对抗损失 + 对齐 L1 损失
```
Loss_G = λ_l2·MSE + λ_lpips·LPIPS + λ_gan·D + λ_align·L1(noise_pred, noise)
```
- λ_l2 = 1.0（像素重建）
- λ_lpips = 5.0（感知损失）
- λ_gan = 0.5（对抗损失）
- λ_align = 1.0（对齐噪声预测）

G/D 交替训练，继承 HYPIR 原始训练流程。
